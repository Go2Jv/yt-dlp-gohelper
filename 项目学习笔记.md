# 项目学习笔记：yt-dlp-gohelper

这份文档用来帮助你把这个项目涉及到的知识点、代码结构、运行方式、扩展方式一次性串起来，方便你后续学习与商业化迭代。

## 1. 这个项目做了什么

`yt-dlp-gohelper` 是一个跨平台的交互式命令行程序（CLI）：

- 自动识别系统：Windows / Linux / macOS（苹果系统）
- 自动识别语言并让用户选择：中文 / English / 日本語
- 启动时检查 `yt-dlp` 与 `ffmpeg` 是否可用
- 缺少依赖时，提供“自动安装/退出手动安装”两种分支
- 依赖确认可运行后，进入交互式下载流程，最终调用 `yt-dlp` 执行下载

本质上：Go 程序负责“用户交互 + 依赖管理 + 拼装参数 + 调用外部命令”，下载能力本身来自 `yt-dlp` 和 `ffmpeg`。

## 2. 目录结构与分层思想

项目把功能按“职责”拆分到不同包（文件夹），这样更容易维护：

- `main.go`：程序入口，只做启动调度
- `app/`：应用编排层（启动流程、分支逻辑）
- `i18n/`：语言与文案（国际化）
- `deps/`：依赖检查与可运行验证
- `install/`：按系统自动安装依赖（使用 build tags 做平台分发）
- `downloader/`：交互式下载流程，负责构建 `yt-dlp` 参数
- `execx/`：执行外部命令的统一封装
- `ui/`：读取用户输入的通用方法

这种拆分的核心好处：

- 入口清晰：`main -> app.Run()`
- 平台差异隔离：`install/` 用 build tag 分开
- 可复用：`execx`、`ui` 这种通用能力可以被多个功能复用
- 易扩展：后续你加“配置文件/非交互参数/多线程队列”等更好加

## 3. 运行流程（从入口到下载）

入口：

- `main.go` 调用 `app.Run()` 作为程序统一入口

启动流程（核心编排）在 `app/app.go`：

1. 检测系统语言环境变量，得到默认语言（`i18n.DetectLanguage`）
2. 提示用户选择语言（`i18n.PromptLanguage`）
3. 输出欢迎信息（包含 OS 显示名，macOS 不显示 darwin）
4. 检测 `yt-dlp/ffmpeg` 是否存在（`deps.Check`）
5. 如果缺失，询问用户选择：
   - 自动安装：`install.Ensure(msg, state)`
   - 退出并提示手动安装：`msg.ManualInstallHint(runtime.GOOS)`
6. 安装后再次验证是否能运行（`deps.VerifyAll`）
7. 进入下载交互（`downloader.RunInteractive`）

## 4. 关键知识点整理（你应该掌握的 Go/工程知识）

### 4.1 Go Module（go.mod）与 import 路径

项目使用 Go Modules 管理依赖与包路径，`go.mod` 的 module 必须是一个“看起来像域名/仓库地址”的路径，例如：

- `github.com/Go2Jv/yt-dlp-gohelper`

这样你在代码里才能正常 `import "github.com/Go2Jv/yt-dlp-gohelper/app"`。

### 4.2 包（package）与文件夹映射

Go 的“包”通常对应一个文件夹：

- `app/app.go` 里声明 `package app`
- 根目录 `main.go` 声明 `package main`

`main` 包必须包含 `func main()`，它是程序启动入口。

### 4.3 跨平台识别：runtime.GOOS

`runtime.GOOS` 会返回当前系统标识：

- Windows：`windows`
- Linux：`linux`
- macOS：`darwin`（Go 内部叫法）

为了界面友好，你在 `i18n.DisplayOSName(goos)` 里把 `darwin` 转成 `macOS / 苹果系统`。

示例（同项目逻辑一致）：

```go
goos := runtime.GOOS
fmt.Println(i18n.DisplayOSName(goos))
```

### 4.4 build tags：按系统编译不同代码

`install/` 里有三套不同系统的安装逻辑，用 build tag 让 Go 在编译时自动选择：

- `install/windows.go` 顶部：`//go:build windows`
- `install/linux.go` 顶部：`//go:build linux`
- `install/mac.go` 顶部：`//go:build darwin`（文件名用 mac.go，但编译条件仍是 darwin）

这样你在 `app` 层就可以统一写：

```go
install.Ensure(msg, state)
```

而不用到处 `if runtime.GOOS == ...` 写分支。

### 4.5 执行外部命令：os/exec

项目的核心能力之一就是调用外部命令（`yt-dlp`, `ffmpeg`, `winget`, `brew`, `apt-get` 等）。

关键点：

- `exec.LookPath("yt-dlp")`：检查命令是否在 PATH 里
- `exec.Command(name, args...)`：用参数切片调用命令（更安全，避免 shell 注入）
- `Stdout/Stderr` 指向 `os.Stdout/os.Stderr`：可以实时输出进度

项目封装在 `execx/execx.go` 里：

- `execx.LookPath`
- `execx.RunInteractive`（透传输入输出）
- `execx.RunCombined`（收集输出，用于检测 `--version`）

### 4.6 环境变量与语言识别

`i18n.DetectLanguage` 从以下环境变量推断语言倾向：

- `LC_ALL`
- `LC_MESSAGES`
- `LANG`

你可以把它理解为：在 Linux/macOS 下，系统通常会设置这些变量，Go 读取它们做出“默认语言”推断。

### 4.7 交互式输入：bufio + os.Stdin

`ui.Prompt` 的常见写法：

- 用 `bufio.NewReader(os.Stdin)` 读取一行
- `strings.TrimSpace` 去掉换行和空格
- 为空时返回默认值

这是写 CLI 的基础能力。

### 4.8 参数拼装：用 slice，而不是拼接命令字符串

在 `downloader` 里，最终是构建 `args := []string{...}`，然后：

```go
execx.RunInteractive("yt-dlp", args...)
```

这种写法对 Windows/Linux/macOS 都一致，而且避免了 `bash -c` 之类的 shell 解析问题，也更安全。

## 5. 怎么使用（普通用户视角）

### 5.1 直接下载编译好的版本

到 Releases 页面下载对应系统的压缩包即可：

https://github.com/Go2Jv/yt-dlp-gohelper/releases

解压后运行可执行文件：

- Windows：双击或在 PowerShell 里运行 `yt-dlp-gohelper.exe`
- Linux/macOS：终端执行 `./yt-dlp-gohelper`

### 5.2 从源码运行（开发者）

```bash
go run .
```

或编译：

```bash
go build -o yt-dlp-gohelper .
./yt-dlp-gohelper
```

## 6. 代码案例：如何做二次开发

### 6.1 增加一种语言（例如韩语）

你需要做两步：

1. 在 `i18n` 里新增语言常量（例如 `Ko`）
2. 扩展所有输出文案分支（欢迎语、提示语、错误语）

经验：国际化最好做到“文案集中管理”，避免散落在各处，否则维护成本会越来越高。

### 6.2 增加一个新的依赖检查（例如 aria2c）

在 `deps` 包里新增字段并检查：

```go
type State struct {
    YtDlp  bool
    Ffmpeg bool
    Aria2c bool
}
```

并在 `Check()` 里加：

```go
_, aErr := execx.LookPath("aria2c")
```

然后扩展 `install` 包按系统安装对应依赖即可。

### 6.3 增加一个下载选项（例如默认开启/关闭 cookies）

可以从 `downloader.RunInteractive` 入手：

- 新增一个 prompt（输入 1/2）
- 根据选择决定是否追加 `--cookies-from-browser ...`

## 7. 注意事项（非常重要）

### 7.1 中国大陆网络环境（强烈建议写在产品文档里）

中国大陆用户下载 YouTube 等外网视频时，推荐使用“全局 VPN（全局模式）”：

- 仅“代理模式/分应用代理”经常会导致 `yt-dlp` / `ffmpeg` 访问失败、速度异常或不稳定

### 7.2 Cookies 读取不稳定（Windows / macOS）

`yt-dlp --cookies-from-browser ...` 在部分系统/浏览器版本上可能失败，常见原因：

- 浏览器正在运行，cookie 数据库被锁定（Permission denied）
- 权限不足、加密实现差异、系统限制等

建议处理方式：

- 彻底关闭浏览器（确保后台进程也退出）后再试
- 或导出 cookies.txt，再用 `yt-dlp --cookies cookies.txt ...`

参考（yt-dlp 官方 issue）：

- Windows 下 Chromium 系列可能出现 Permission denied：https://github.com/yt-dlp/yt-dlp/issues/7271
- Windows 下 Firefox cookies.sqlite Permission denied：https://github.com/yt-dlp/yt-dlp/issues/15760

### 7.3 PATH 问题（尤其 Windows）

你安装完 `yt-dlp/ffmpeg` 后，必须保证在终端能直接运行：

```text
yt-dlp --version
ffmpeg -version
```

如果新装完仍然提示找不到命令：

- 重新打开一个终端（让 PATH 刷新）
- 确认安装器是否真的把路径写进了系统 PATH

## 8. 你可以继续怎么“商业化”升级（学习路线建议）

按优先级建议你后续做这些升级（也是你学习的路线图）：

1. 非交互模式：支持命令行参数（例如 `--url --out --quality`），方便脚本化/GUI 调用
2. 配置文件：保存默认下载路径、默认分辨率、默认 cookies 策略
3. 更清晰的错误码与日志：便于售后/客服排查
4. 自动发布流水线：GitHub Actions + 自动打包 + 自动 Release（减少人工发布）
5. GUI：Tauri/Electron/Fyne/Wails 等做桌面端（看你的目标用户）
